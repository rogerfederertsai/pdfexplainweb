# 地址 OCR 校正映射表

本檔案定義 EasyOCR 在辨識圖片加密地址時，常見的字元誤辨對照。  
main.py 會於啟動時自動載入此表，無需修改程式碼即可新增或調整映射。

## 使用說明

- **格式**：每行一個映射，`錯誤字元 -> 正確字元`，`#` 開頭為註解。  
- **新增映射**：直接在下方表格新增一行即可，重新執行程式即生效。  
- **注意**：映射為「字元替換」，請勿使用過於通用的字（如「的」、「一」）以免誤替換。

## 映射規則

```
# 臺字系列（OCR 常將臺誤讀為以下字）
耋 -> 臺
耸 -> 臺
壹 -> 臺

# 學字系列
孿 -> 學
孽 -> 學
壆 -> 學
荸 -> 學

# 其他常見地址用字誤讀
覃 -> 南
芸 -> 巷
邾 -> 鄉
麴 -> 鄰

# 閩南語地址字（厝）
# 盾 與 厝 在低解析度圖片中易被混淆
盾 -> 厝

# 道路名稱誤讀（民椿路 / 民榨路 → 民權路）
# 椿、榨 與 權 在某些字型或解析度下相似
椿 -> 權
榨 -> 權

# 村落名稱誤讀
# 飛 與 雅 在低解析度圖片中易被混淆（仁飛村 → 仁雅村）
飛 -> 雅

# 村字系列（杠 與 村 易混）
杠 -> 村

# 遮罩字元誤讀（地址中的 *** 隱碼在高 DPI 下有時被讀為「咒」）
咒 -> ***
```

## 正規表達式後處理規則

以下規則以正規表達式形式處理，無法以單一字元映射解決的情況：

```
# 數字結尾的英文 l 或 I（大寫 i）→ 號
# 例：156l → 156號、3I → 3號
REGEX: (\d+)\s*[lI]$ -> \1號

# 數字後的英文 h/H 接漢字，可能是「鄰」的 OCR 誤讀
# 例：1h中崙 → 1鄰中崙
REGEX: (\d)h([^\d\s]) -> \1鄰\2
REGEX: (\d)H([^\d\s]) -> \1鄰\2

# 遮罩字元（***）前若有多餘空白，予以清除
# 例：北興街 *** → 北興街***
REGEX: \s+(\*{2,}) -> \1
```
